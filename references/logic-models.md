# Logic Models

Use these models to make the planning brief more rigorous. Name the model used in the relevant section.

## 1. Audience-Shift-Action Model

```text
Audience -> Current belief -> Desired judgment -> Target action
```

Use in task diagnosis. If any part is vague, ask questions before continuing.

## 2. Resistance Map

```text
What the audience doubts -> Why they doubt it -> What proof can reduce doubt -> What action remains realistic
```

Use for sales, decision, pitch, and internal change decks.

## 3. Evidence Pyramid

Rank available support from strongest to weakest:

1. verified data with clear caliber
2. real screenshots or original records
3. customer/user quotes with authorization
4. concrete case process
5. expert/industry reference
6. logical inference
7. concept image or emotional expression

Do not let weak evidence carry a high-risk conclusion.

## 4. Feature-To-Problem Translation

For sales and product decks:

```text
feature -> customer problem -> business consequence -> page expression -> proof
```

Avoid starting with product functions before the audience recognizes the problem.

## 5. Cognition Route

```text
They currently believe -> first they must see -> then they must admit -> then they can accept -> finally they can act
```

Use for mainline and page ordering.

## 6. Core Contradiction Model

```text
Goal / current bottleneck / cost of not changing / reason current solution fails / proposed shift
```

Use to find the mainline sentence.

## 7. Narrative Structure Selector

- Sales: problem progression or trust building.
- Report: conclusion first.
- Decision: option comparison and recommendation.
- Training: task training.
- Pitch: trust building.
- Communication: memory point.
- Data report: question-to-insight.

If the PPT is mixed, pick a primary structure and mark secondary needs.

## 8. Slide Task Canvas

For every slide:

```text
Page task / judgment moved / conclusion title / evidence / visual form / speaker note / transition
```

If a page has no task, remove or merge it.

## 9. Page Copy Compression Model

Rewrite in three levels:

1. label: `市场背景`
2. judgment: `客户增长放缓来自渠道质量下降`
3. action-oriented judgment: `下月应从追线索数量转向优化渠道质量`

Prefer level 2 or 3.

## 10. Data Insight Ladder

```text
data point -> pattern -> interpretation -> business judgment -> action
```

Use for every data page. A chart without action is not enough.

## 11. Asset Priority Matrix

Choose assets by purpose:

- Build trust: real screenshots, data, case evidence.
- Explain structure: process diagram, comparison table, matrix.
- Teach operation: step screenshot, exercise sample, before/after.
- Create memory: concept visual, story image, keynote visual.

Use the weakest asset only when stronger assets do not exist and the risk is low.

## 12. Delivery-Shape Decision Matrix

Use before image production:

```text
reuse need / likely edits / data-update frequency / source-file promise / deadline / budget -> delivery shape
```

Decision rules:

- One-time preview: quick visual draft or image-version PPT.
- Most commercial drafts: semi-editable, with text rebuilt outside images.
- Reused templates, pitch decks, consulting reports, corporate materials: editable/source reconstruction.
- Frequently updated charts: rebuild as real charts.
- High-value visual pages: consider asset-package reconstruction.

## 13. Page Production Table Model

For every page:

```text
page type / page purpose / title / on-page text / main visual / layout freedom / text strategy / delivery shape / risk
```

This prevents AI from guessing and makes later revision traceable.

## 14. Five-Round Review Model

Review in this order:

1. Target: does the deck complete the original task?
2. Logic: can the deck be connected by a "because -> therefore" chain?
3. Fact: are data, names, cases, policies, and AI visuals defensible?
4. Expression: is each page clear, readable, and speakable?
5. Delivery: can the recipient open, present, edit, and reuse the files as promised?

Do not start with visual polish if target, logic, or fact checks fail.

## 15. Risk Register

For each high-risk claim:

```text
claim / current evidence / missing proof / wording risk / safer expression / owner to verify
```

Include the risk register in the final brief.
