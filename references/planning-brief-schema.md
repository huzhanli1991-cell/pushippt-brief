Final quality rules:

- Every page must have one task and one conclusion.
- Every important claim must have evidence or a risk note.
- Use proactive web research when public facts, market data, product information, policies, or comparable cases can improve the content; cite sources in `资料来源表.md`.
- Do not use source-unclear or unverifiable information as evidence.
- Data pages must include source/caliber or be marked as pending.
- Visual and asset suggestions must be specific enough for production.
- Style recommendation must use exact field names from bundled `references/PPT制作风格库.md`.
- Page production table must include usage mode, text density, required in-image content, must-keep facts, design freedom, and delivery shape.
- PushiPPT production must call imagegen directly for each confirmed page and render required content into the generated image.
- Imagegen prompts must prioritize visual effect and allow layout/content organization freedom except for must-keep facts.
- Generated images must not include page numbers, logos, watermarks, or footers.
- Product visuals must stay consistent across pages when products appear.
- Inner-page titles should keep size and position consistent; cover title should receive special beautified treatment.
- After page images are generated, create an image-version PPTX by inserting images in content order.
- The complete manuscript must be written and saved as Markdown only after page images are generated.
- Do not promise editable PPTX/source reconstruction unless the user explicitly asks for a reconstruction workflow.
- After the saved brief and page production table are confirmed, follow `references/production-workflow.md`.
